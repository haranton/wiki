Индекс - это структура данных для ускорения поиска данных в базе данных.

Виды индексов:

1. B-tree операции  = , >, \< , \<=, >= ,  LIKE abc% O(log N)

2. Hash (не рекомендуется) операции = O(1)

3)GIN операции %abc%

4)GIST операции %abc%



Плохая сторона индексов что они уменьшают скорость операций DELETE, UPDATE, INSERT, тк нужно балансировать и поддерживать структуры данных для ускоренного поиска

b tree - это сбалансированное дерево есть ключи (по ним идет поиск) и ветви сам значения (указатели на строки в базе данных) сложность log n
