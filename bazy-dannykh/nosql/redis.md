Redis - это база данных no sql которая хранится в оперативной памяти, что дает огромную скорость работы операций. 

Цели: 

1) Кеширование данных, чтобы не обращаться к бд и получить данные быстрее

2) Брокер сообщений (в виде очереди)



Виды данных хранения 

| Тип данных                                     | Ключевые команды                           | Описание                                                       | Пример использования                                     |
|------------------------------------------------|--------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------|
| **String** (строка)                            | `SET`, `GET`, `INCR`, `DECR`               | Простое значение (текст, число, JSON, бинарные данные).        | Кэш JSON-ответа, токен авторизации, счётчик просмотров.  |
| **List** (список)                              | `LPUSH`, `RPUSH`, `LPOP`, `RPOP`, `LRANGE` | Упорядоченный список строк (аналог очереди или стека).         | Очередь задач, история действий пользователя.            |
| **Set** (множество)                            | `SADD`, `SREM`, `SMEMBERS`, `SISMEMBER`    | Неупорядоченное множество уникальных элементов.                | Хранение уникальных тегов, списков пользователей онлайн. |
| **Sorted Set (ZSet)**(упорядоченное множество) | `ZADD`, `ZRANGE`, `ZREVRANGE`, `ZREM`      | Множество с числовым "весом" (score), по которому сортируется. | Рейтинги, таблицы лидеров, статистика популярности.      |
| **Hash** (хэш-таблица)                         | `HSET`, `HGET`, `HGETALL`, `HDEL`          | Ассоциативный массив (ключ -- поле -- значение).               | Хранение профиля пользователя, объекта товара.           |
| **Bitmap** (побитовые операции)                | `SETBIT`, `GETBIT`, `BITCOUNT`             | Массив битов -- эффективно хранит true/false.                  | Учёт посещений по дням, активность пользователей.        |
| **HyperLogLog**                                | `PFADD`, `PFCOUNT`                         | Приближённый подсчёт уникальных элементов (экономия памяти).   | Подсчёт уникальных IP, просмотров, пользователей.        |
| **Stream**                                     | `XADD`, `XREAD`, `XGROUP`                  | Лог событий / поток данных (аналог Kafka topic).               | Потоки логов, сообщения IoT, обработка событий.          |
